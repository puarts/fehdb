# freezedetect の min_duration 閾値問題

## 概要

`freezedetect` の `min_duration`（`d`パラメータ）が 1.5秒 の場合、一部のスキル画面が取りこぼされる。

## 発生状況

- 動画: 10-03-16（新英雄 響心エリウッド＆つながりニニアン）
- 対象フレーム: 「習得可能スキル (4/5)」画面（戦友との絆の斧 + 集いし炎）
- strict freezedetect (noise=0.003) での静止区間: 30.33s〜31.80s（**1.47秒**）
- `d=1.5` で検出されず、`d=1.25` で検出される

## 原因

テキスト量が多いスキル画面の表示時間が短く（約1.47秒）、`d=1.5` の閾値をわずかに下回る。

## 暫定対応

`min_duration` のデフォルトを **1.4秒** に変更。
1.47秒のスキル画面は拾えるが、将来さらに短い表示時間のケースがあり得る。

## d=1.25 にした場合の問題

「！」マーク（新スキルバッジ）のアニメーションにより静止区間が分割され、同じスキル画面が重複検出される。d=1.25 では「！」アニメーション間の区間（約1.27秒）も拾ってしまう。

## 根本的な解決案

1. **重複除去の強化**: d を下げつつ、パーセプチュアルハッシュの重複除去で同一画面を統合
2. **「！」アニメーション対応**: freezedetect 後に近接する同一画面の区間を結合（マージ）
3. **adaptive duration**: 動画の特性に応じて d を自動調整
